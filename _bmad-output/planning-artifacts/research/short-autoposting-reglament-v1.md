# Короткий регламент автопубликации (v1)

Дата: 2026-03-05  
Проект: public-bmad

## 1. Базовые правила

1. Публикуем только через официальные API/официальные интеграции (OAuth).
2. Не публикуем один и тот же ролик одновременно на 5 аккаунтов.
3. Для каждого аккаунта делаем вариации: заголовок, описание, обложка, CTA.
4. На всех аккаунтах включены `2FA`, пароли не передаются сторонним сервисам.

## 2. Лимиты на старт (первые 14 дней)

1. `1–2` публикации в день на аккаунт.
2. Между публикациями одного и того же ролика на разных аккаунтах: минимум `60–180 минут`.
3. Не делать резких скачков объема (рост не больше `+20–30%` в неделю).

## 3. Ежедневный процесс

1. Проверка перед запуском: нет предупреждений платформ, нет аномальных входов, API-лимиты в норме.
2. Генерируем 5 версий метаданных на платформу (без копипаста).
3. Публикуем по расписанию с очередью и retry (`exponential backoff`).
4. После публикации фиксируем статус: `ok / warning / failed` + код ошибки.

## 4. Стоп-правила (аварийная пауза)

1. `>=3` ошибок `429/4xx` подряд на одной платформе.
2. Любой security challenge/подозрение на взлом.
3. Любое policy warning по спаму/automation.
4. При стопе: отключаем автопостинг на платформе на `24 часа`, разбираем причину, затем мягкий рестарт.

## 5. Еженедельный контроль

1. Смотрим: охваты, частоту ошибок, предупреждения.
2. Если всё стабильно 2 недели, увеличиваем нагрузку постепенно.
3. Если метрики падают, сначала снижаем синхронность и усиливаем вариативность контента, потом уже меняем объем.
